Detailed Description

A simple controls node.

Author
Nelson Durrant
Date
September 2024

This node subscribes to desired depth, heading, and speed topics and actual depth and heading topics. It then computes the control commands using various controllers and publishes the control commands to a control command topic.

Subscribes:

  • init (std_msgs/msg/Empty)
  • desired_depth (frost_interfaces/msg/DesiredDepth)
  • desired_heading (frost_interfaces/msg/DesiredHeading)
  • desired_speed (frost_interfaces/msg/DesiredSpeed)
  • depth_data (geometry_msgs/msg/PoseWithCovarianceStamped)
  • modem_status (seatrac_interfaces/msg/ModemStatus) Publishes:
  • controls/command (frost_interfaces/msg/UCommand)

Control command publisher.

This publisher publishes the control commands to the "control_command" topic. It uses the UCommand message type.

Initialization subscriber.

This subscriber subscribes to the "init" topic. It uses the Empty message type.

Desired depth subscriber.

This subscriber subscribes to the "desired_depth" topic. It uses the DesiredDepth message type.

Desired heading subscriber.

This subscriber subscribes to the "desired_heading" topic. It uses the DesiredHeading message type.

Desired speed subscriber.

This subscriber subscribes to the "desired_speed" topic. It uses the DesiredSpeed message type.

Depth subscriber.

This subscriber subscribes to the "depth_data" topic. It uses the PoseWithCovarianceStamped message type.

Yaw subscriber.

This subscriber subscribes to the "modem_status" topic. It uses the ModemStatus message type.

Control timer.

This timer calls the control loop at the specified period.
